Problem Statement

Linked Lists are typically composed of node objects chained together via node pointers. Each node has a next pointer which points to the next node in the chain. Since nodes are almost always allocated on the heap, it is important to clean up nodes using delete when done with them. This is typically accomplished by finding the last node in the linked list, deleting it, and working backwards. The last node is typically identified by having a NULL next pointer.

However, a problem arises if the linked list has become corrupted in some way. One situation that can cause trouble is if a cycle has been produced in the linked list. A cycle occurs when some node in the linked list links back to a previous node in the linked list. In this case, detecting the last node by looking for a NULL next pointer is impossible. An attempt to do so will result in an infinite loop, running around and around the loop (or cycle) in the linked list.

In this lab, you will detect cycles in linked lists and find out their length. The length of a cycle is defined as the number of nodes participating in the cycle. The smallest cycle is one where a node points to itself (thus having a length of 1).

Create a function cycle_length, which takes in a pointer to the head of the linked list, and, if there is a cycle in the linked list, returns the length of the cycle. If there is no cycle, your function should return -1.

Notes and Constraints

  • The linked list starting at head contains between 1 and 25 nodes, inclusive.
