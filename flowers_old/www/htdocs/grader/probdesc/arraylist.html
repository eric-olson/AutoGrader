Lab: Array List

Specification

   class list {
   public:
	   list();	             
	   list(const list&);     
	   ~list();

	   void operator =(const list&);

	   int size();
	   int get(int position);
	   void add(int);
	   void insert(int n, int position);
	   void erase(int position);
   };

Problem Statement

You are to create a functioning list class using dynamically allocated arrays for storage. This list class will store integers only. The required public functions are shown in the specification block. Note that these are a minimum requirement, and you can add any public or private member functions or variables as you need to make these public functions work as specified.

The required public member functions are:

  • list() - the default constructor.
  • int size() - returns the current count of elements stored in the list.
  • int get(int position) - returns the element stored at index position; if position is greater than the size of the list, returns 0.
  • void add(int) - adds an element to the end of the list.
  • void insert(int n, int position) - inserts an element before the element at index position. E.g., if the list is 1 2 3 4, then insert(17,2) results in 1 2 17 3 4. The method does nothing if position is greater than the size of the list.
  • void erase(int position) - removes the element at index position. E.g., if the list is 1 2 3 4, then erase(2) should result in 1 2 4. The method does nothing if position is greater than or equal to the size of the list.

In addition, for full credit you will need to implement the "big 3":

  • list(const list&) - the copy constructor.
  • ~list() - the destructor
  • void operator =(const list&) - the assignment operator.

Be sure to use incremental development on this task. A majority of the tests can be passed by simply correctly implementing a list class which holds a maximum of 4 elements. Most of the tests can be passed without a properly functioning copy constructor and assignment operator. So work on the basic functionality first, then expand as you go.

Test often! On this problem it is easy to generate memory exceptions, which provide you with minimal test feedback. So try to find your memory errors as soon as they occur and fix them before coding additional functions (otherwise you might generate additional memory errors, but not know where to look for them!)
